ライブ分析のレビュー

このトピックでは、Live モジュールによって提供されるライブ分析について説明します。

概要

ライブモジュールは、ライブストリーミングイベントまたはチャンネルのパフォーマンスに関するレポートを提供します。リアルタイム同時視聴者と視聴者のストリーム時間、地理、およびデバイスに関する集計されたイベント後データを含む情報を利用できます。

ライブ分析を表示するには、次の手順に従います。

  1. 開くライブモジュール。
  2. イベント名をクリックします。
  3. クリック報告左のナビゲーションにあります。

分析ダッシュボードは、次のセクションで構成されています。

  • スナップショット
  • デバイス別表示回数
  • 国別表示回数

スナップショット

スナップショットセクションが表示されます

  • 総視聴回数- ライブ ストリーム期間中の総視聴回数。
  • ユニーク視聴者数- ストリームを視聴しているユニーク視聴者の総数。たとえば、同じユーザーが 1 時間以内に 2 回別々のライブストリームに接続すると、1 つの一意のビューアとしてカウントされます。
  • 平均視聴時間- 特定のセッションの平均消費時間。たとえば、ユーザー A が 1 分間のコンテンツを監視し、ユーザー B が 2 分間のコンテンツを監視する場合、平均表示時間は 1.5 分です。
  • 総視聴時間- に表示されるストリームの期間中に費やされた合計時間時:分:秒.
  • 広告開始- 広告開始数

スナップショットの下に、同時閲覧者を示すグラフが表示されます。同時閲覧者は、位置に関係なく、任意の時点でストリームに接続されたセッション数を測定します。たとえば、10 人のユーザーがライブストリームをライブエッジで視聴していて、5 人のユーザーが DVR ウィンドウ（ライブの 10 分遅れ）で見ている場合、1 分間の同時視聴者は 15 ユーザーになります。

デバイス別表示回数

[ViewsbyDevice]セクションには、デバイスごとのビュー数を示すグラフが表示されます。デバイスタイプは次のとおりです。

  • デスクトップ- デスクトップおよびラップトップ コンピュータを含む
  • タブレット- iPad、Android タブレットなどが含まれます。
  • モバイル- 携帯電話
  • コネクテッド TV/ゲーム機/コンパニオン デバイス- スマート TV、セットトップ ボックス、ゲーム コンソールを含む
  • 他の- 上記のいずれにもマッピングされていないユーザー エージェントの場合

国別表示回数

国別ビューセクションには、国別のビュー数のグラフが表示されます。

レポートデータのエクスポート

をクリックすると、ライブ イベント レポート データを Microsoft Excel ファイルにエクスポートできます。 XLSXボタン。

注

ライブ分析に関する一般的な注意事項:

  • 分析は、ライブイベントで生成されたデフォルトのライブビデオIDに対して記録されます。顧客がm3u8プレイリストを再生用に取得するか、それを使用してリモートアセット（新しいビデオID）を作成した場合、新しいアセットの分析は収集されません。
  • それだけ同時視聴者リアルタイムで表示されます。他のメトリックは、標準の分析と同様の遅延で収集および集計されます
  • 同時視聴者Brightcove Player および SDK (または Brightcove Data Collection API を使用する非 Brightcove Player) で配信されるストリームに対してのみ正確です。
  • データはイベント終了後最大32日間利用可能です
  • リアルタイムは、1分間隔で集約されたセッションによって定義されます
  • 定期的なイベント (静的エントリポイント) では、最新のアクティブ化の開始から分析が提供されます
  • 現時点では、分析モジュールは正確な一意のビューアデータを取得できず、LIVE モジュール分析の間に矛盾があります。ライブアナリティクス UI を使用して、視聴者の数を確認してください。
  • 以下に示すように、特定の広告ブロッカーは、ライブ分析データが正しく読み込まれないようにします。

    広告ブロッカーを無効にすると、問題が解決します。
